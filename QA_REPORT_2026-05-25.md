## Bugs

### Bug #1 — Block/Report action sheet breaks out of the app frame
**Severity:** Medium (web-only, cosmetic, but looks broken)
**Where:** "..." menu on MatchDetail screen, and "..." menu inside a 1:1 Chat
**What happens:** The Block/Report action sheet renders at the full desktop browser width (~1470px) instead of being constrained to the ~440px mobile app column. It visually spills outside the app.
**Repro:**
1. Open the web app on desktop
2. Tap any person → MatchDetail → tap "..." (top right)
3. Sheet appears stretched across the whole browser window
4. Same bug on the "..." menu inside a direct chat
**Impact:** Native iOS/Android builds won't show this (the viewport *is* phone width). Only desktop-web users see it. Since found-community.vercel.app is your public web presence, it looks unpolished.
**Likely cause:** The action sheet container isn't inheriting the app's max-width wrapper — it mounts to the root instead of the phone-frame container.

### Bug #2 — Group member count is inconsistent across screens
**Severity:** Low–Medium (wrong number shown to users, no functional break)
**Where:** Groups feed cards vs. Group detail page
**What happens:** Same group reports different member counts depending on the screen.
**Repro (observed on "Walkthrough Test Group"):**
1. Groups tab — Suggested card showed **"1 member"**
2. Tapped Join — the Joined card then showed **"3 members"**
3. Opened the group — detail page showed **"2 members"** (the correct number: owner + me)
4. Left the group — detail correctly dropped to "1 member"
**Impact:** The detail page is accurate. The feed GroupCard count is unreliable — likely a row-multiplying SQL JOIN or a stale cached value in `my_groups_feed()`. Users will see wrong counts on the browse list.

## Observations (not bugs — but read these before launch)

1. **The app looks empty.** Discover shows only 1 person. Activity has 0 inbound. This is thin seed data, not a bug — but a brand-new user opening FOUND will see a near-dead app. **This is your real launch risk.** Before promoting: seed a believable set of profiles per launch market, or stage the launch so the first cohort lands together. An empty social app reads as "broken."

2. **Couldn't fully test 3 things** due to thin data / browser limits, all because there was nothing to act on — not because they failed:
   - Wave button (no idle person card available to wave at)
   - Accept / Dismiss inbound connections (Activity tab was empty)
   - Photo upload (avatar, highlight reel, group cover) — opens a native file picker that can't be driven from automated testing. UI elements are all present.

3. **Two near-identical test accounts** exist — "ryder schilling" (logged in) and "Ryder Schilling" (seed account you're connected to / who owns Walkthrough Test Group). Harmless, but confusing. Clean up test accounts before launch.

4. **Leftover from this QA pass:** I created "QA Test Group" (Inlet Beach, FL — description says "safe to delete") to test the create-group flow. Delete it when convenient.
